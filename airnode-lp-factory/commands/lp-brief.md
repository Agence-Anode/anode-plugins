# Skill /lp-brief — Redaction brief enrichi

## Usage
```
/lp-brief [client]
```

## Pre-requis

La strategie copy doit etre validee. Lire :
- `~/Documents/[client]/assets/config-client.md`
- `~/Documents/[client]/assets/branding.md`
- Tous les fichiers dans `~/Documents/[client]/assets/recherches/`
- `~/Documents/airnode/templates/brief-template.md`

## Instructions

Tu rediges le brief enrichi complet en suivant le template.

### Etape 1 — Lire le template

Lire `~/Documents/airnode/templates/brief-template.md` pour le format exact.

### Etape 2 — Rediger

Remplir TOUTES les sections avec du contenu reel. Aucun placeholder, aucun "[a completer]".

Sources pour chaque section :
- **Metadata** : infos de `/lp-init`
- **Branding** : `assets/branding.md`
- **Positionnement** : `recherches/offre-positionnement-*.md`
- **Enjeux** : `recherches/contexte-marche-*.md` (pain points)
- **Solution** : `recherches/offre-positionnement-*.md`
- **Features** : `recherches/offre-positionnement-*.md`
- **Avantages competitifs** : `recherches/benchmark-concurrents-*.md`
- **Social proof** : `recherches/social-proof-*.md`
- **Messages cles** : `recherches/strategie-copy-*.md` (hero validee)
- **FAQ** : synthese des objections de `recherches/contexte-marche-*.md`
- **Contexte campagne** : infos consultant de `/lp-init`

### Etape 3 — Checklist

Avant de presenter au consultant, verifier :
- [ ] Metadata complete
- [ ] Branding renseigne
- [ ] Toutes les sections avec contenu reel
- [ ] Headline, sous-titre, CTA rediges
- [ ] Au moins 1 cas client / temoignage
- [ ] Au moins 3 features avec benefice
- [ ] Contexte campagne renseigne

### Etape 4 — Validation

Sauvegarder dans `~/Documents/[client]/briefs/declinaisons/[slug].md`

Presenter le brief au consultant pour relecture. Demander : "Le brief est complet, tu veux ajuster quelque chose avant de passer au wireframe ?"
